🔑 Authentication

The AICore API gateway requires authentication for all API endpoints. You must provide a valid API Key obtained from your Developer Console. Keys can be supplied using either of the following standard headers:

x-api-key Custom Header

Pass your token as a header parameter:
x-api-key: mk_live_5893a7d2e...

Authorization Bearer Token

Pass your token using the standard authorization format:
Authorization: Bearer mk_live_5893a7d2e...

⚡ Quickstart

Run a simple cURL command to verify your access. Make sure to replace YOUR_API_KEY with the actual key from your dashboard console:

Verify API Connection (cURL) 
curl -X POST http://localhost:3001/api/chat \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Verify connection. Reply with \"API online\".",
    "model": "k2d6",
    "stream": false
  }'

💬 Chat Completions

Route prompts and configure output formatting through our stateless or stateful chat completion gateway.

Endpoint: POST /api/chat

Request Body Parameters

The API handles automated YAML frontmatter injection. Pass any of the parameters below in the JSON request body to shape the model output:

Parameter Type Status Description
message string Required The main text prompt or instruction sent to the model.
model string Optional Model key to use. Defaults to k2d6.
stream boolean Optional Enable Server-Sent Events (SSE) streaming. Defaults to true.
instructions string Optional Base prompt guidelines. Automatically suffix-locked with #exact-pattern.
role string Optional Specifies the persona or profession the model should adopt.
tone string Optional Tone of response (e.g. technical, enthusiastic, academic).
audience string Optional Intended reader (e.g. beginners, CTOs, executive board).
format string Optional Desired format (e.g. markdown, json, raw).
structure string Optional Layout configuration (e.g. introduction-body-conclusion).
max_words number Optional Hard boundary count for response length.
no_emojis boolean Optional Instructs the model to completely omit emojis from the response.
bullets_only boolean Optional Instructs the model to respond strictly with a list of bullet points.

🌊 SSE Streaming Integration

By enabling "stream": true, the gateway responds using standard Server-Sent Events (SSE). This is ideal for chat UIs, delivering real-time responses with sub-second initial token latency.

When using SSE streaming, the API returns the text/event-stream content type. Each chunk is prefixed with data:, and a final data: [DONE] message is emitted when the generation is complete.

JavaScript Stream Consumer Example

Frontend SSE Reader Script 
async function streamKimi() {
  const response = await fetch('http://localhost:3001/api/chat', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Accept': 'text/event-stream',
      'x-api-key': 'YOUR_API_KEY'
    },
    body: JSON.stringify({
      message: 'Write an essay about space exploration.',
      stream: true
    })
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder('utf-8');
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop(); // save incomplete line

    for (const line of lines) {
      const cleanLine = line.trim();
      if (!cleanLine.startsWith('data:')) continue;

      const dataStr = cleanLine.substring(5).trim();
      if (dataStr === '[DONE]') {
        console.log('\n--- Stream Complete ---');
        return;
      }

      try {
        const parsed = JSON.parse(dataStr);
        if (parsed.delta) {
          process.stdout.write(parsed.delta); // Print token real-time
        }
      } catch (err) {
        // Skip formatting lines
      }
    }
  }
}

⚠️ Response Status & Error Codes

The gateway maps Kimi response patterns to standard HTTP status codes. Refer to the error catalog below for handling failures gracefully:

HTTP Code Error String Resolution
401 Authentication required The request header lacked a valid x-api-key or Authorization bearer token.
403 Quota exhausted Your developer key has consumed its full allocated request volume. Please upgrade or contact support.
400 Missing or invalid message The JSON payload didn't contain a valid "message" parameter in the request body.
500 Internal API Error An issue occurred with the upstream Kimi model engine or Puppeteer session rotator. The gateway auto-recovers sessions dynamically in the background.
