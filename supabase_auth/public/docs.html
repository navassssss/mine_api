Authentication

To authenticate requests, obtain your API Key from the Developer Console and pass it in either of the following formats:

x-api-key Header Parameter

Pass your developer token as a request header: x-api-key: your-api-key.

Authorization Header Parameter

Pass your developer token as a standard Bearer authorization header: Authorization: Bearer your-api-key.

Chat Completions

Send chat prompts and query parameters to the chat completion gateway. Supports SSE streaming and standard responses.

Endpoint: POST /api/chat

message string (required)

The prompt message string to send to the Kimi intelligence engine.

model string

Model key to route requests: k2d6, k2d6-thinking, k2d6-agent, k2d6-agent-ultra.

YAML Formatting System

You can customize structural outputs by adding these keys to your request body. The gateway wraps them into Kimi-readable YAML frontmatter automatically:

instructions string

System instructions to force specific persona behaviors or restrict formatting patterns on the model.

max_words number

Strict maximum boundary limit of tokens or words returned in response.
